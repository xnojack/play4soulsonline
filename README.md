# Four Souls Online

An unofficial self-hostable web companion for playing The Binding of Isaac: Four Souls tabletop game online. Players connect to a shared game room and play with a fully synchronized board — cards, decks, the stack, priority, health trackers, and more.

## Disclaimer

This project is 100% vibe coded. Not because I love AI, or think AI is great, but because I don't have the time to write and maintain another passion project when my other coding projects are all currently dead. My friends and I just want to play 4 souls online outside of TTS.

## Running with Docker

The easiest way to self-host. No Node.js required — just Docker and Docker Compose.

### 1. Create a `docker-compose.yml`

```yaml
services:
  server:
    image: ghcr.io/xnojack/play4soulsonline/server:latest
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data
    environment:
      - MAX_ROOMS=10
    restart: unless-stopped

  client:
    image: ghcr.io/xnojack/play4soulsonline/client:latest
    ports:
      - "3000:80"
    environment:
      - VITE_SERVER_URL=       # leave empty — nginx proxies to the server internally
      - VITE_PUBLIC_URL=       # optional: your public URL for OG/social previews
    depends_on:
      - server
    restart: unless-stopped
```

### 2. Scrape card data

This downloads all 1081+ card images and data from [foursouls.com](https://foursouls.com) into a local SQLite database. Only needs to be run once. Takes 10–20 minutes (rate-limited to be polite to their servers). If interrupted, re-run the same command — the scraper is incremental.

```bash
docker run --rm -v ./data:/app/data ghcr.io/xnojack/play4soulsonline/scraper:latest
```

Card images are saved to `./data/cards/` and the database to `./data/cards.db`.

### 3. Start

```bash
docker compose up -d
```

Open http://localhost:3000.

## Building Locally

For contributors or anyone who wants to run from source.

### Prerequisites

- Node.js 18+
- npm 9+

### 1. Install dependencies

```bash
npm install
```

### 2. Scrape card data

```bash
make setup
# or: npm run scrape
```

### 3. Start development servers

```bash
make dev
# or: npm run dev
```

- Client: http://localhost:3000
- Server API: http://localhost:3001

### Building production images locally

```bash
cp .env.example .env
# Edit .env if needed
make up-build   # docker compose up --build
```

## Configuration

Set these as environment variables in your `docker-compose.yml` or `.env` file.

| Variable | Default | Description |
|---|---|---|
| `SERVER_PORT` | `3001` | Host port for the game server |
| `CLIENT_PORT` | `3000` | Host port for the web client |
| `MAX_ROOMS` | `10` | Max concurrent game rooms |
| `VITE_SERVER_URL` | _(empty)_ | URL the browser uses to reach the server. Leave empty when using the default Docker Compose setup — nginx proxies internally. Only set this if the server is on a different domain. |
| `VITE_PUBLIC_URL` | _(empty)_ | Your public-facing URL. Used for OG/Twitter social preview meta tags only. |
| `ROOM_TIMEOUT_LOBBY_MS` | `300000` | How long to keep a room alive after all players disconnect in the lobby (5 min) |
| `ROOM_TIMEOUT_ACTIVE_MS` | `1800000` | How long to keep a room alive after all players disconnect mid-game (30 min) |
| `ROOM_TIMEOUT_ENDED_MS` | `600000` | How long to keep a room alive after the game ends (10 min) |
| `ROOM_TIMEOUT_CREATED_MS` | `1800000` | How long to keep a room alive if nobody ever joins (30 min) |

## How to Play

1. One player clicks **Create Game** and shares the 6-character room code.
2. Other players click **Join Game** and enter the code + their name.
3. Spectators can join with "Join as spectator" checked.
4. The host selects which card sets to include and clicks **Start Game**.

### In-game controls

- **Cards** — hover for a quick tooltip; click for full card details.
- **Attack** — click "Attack" on a monster slot during your turn. A dice roller appears. Click it to roll; the attack resolves automatically.
- **Loot cards** — click a card in your hand, then click "Play" (costs a loot play).
- **Items** — click your own items to toggle charged/deactivated state.
- **Coins / HP** — use the +/- buttons next to each stat.
- **Stack** — all loots and abilities appear in the stack panel. Click "Pass Priority" to pass. When all players pass, the top item resolves.
- **Hand sharing** — click "Share Hand" in your player area and pick a player.
- **Card Search** — click "Card Search" in the top bar to look up any card mid-game.

## Architecture

```
packages/
  scraper/    # One-time card scraper (Node.js + Cheerio)
  server/     # Game server (Express + Socket.io + SQLite)
  client/     # Web client (React + Vite + Tailwind + Zustand)
data/
  cards.db    # SQLite card database (generated by scraper)
  cards/      # Card images (generated by scraper)
```

### Server

- All game state lives in memory (`GameStore`).
- Each action is a socket event; the server mutates state and broadcasts a filtered `ClientGameState` to every player in the room.
- Hands are private: the server only sends hand card IDs to the player who owns them (or players they've shared with).
- Card effects are **manual** — the app tracks game state but players decide how card text is applied.

### Stack & Priority

Priority passes around the table. Any player with priority can play a loot, activate an ability, or pass. When all players pass in succession, the top item on the stack resolves. This matches the Four Souls extended rulebook exactly.

## Re-scraping / Updating Cards

To add new card sets or refresh card data:

```bash
# Docker
docker run --rm -v ./data:/app/data ghcr.io/xnojack/play4soulsonline/scraper:latest

# Local
npm run scrape
```

The scraper is incremental — it skips cards already in the database and only downloads new ones.
