# Four Souls Online

A self-hostable web companion for playing The Binding of Isaac: Four Souls tabletop game online. Players connect to a shared game room and play with a fully synchronized board — cards, decks, the stack, priority, health trackers, and more.

## Disclaimer

This project is vibe coded. Not because AI is great, but because I don't have the time to write and maintain another passion project when my other coding projects are all dead. It works well enough. PRs welcome.

## Quick Start

### Prerequisites

- Node.js 18+  
- npm 9+  
- (Optional) Docker + Docker Compose for containerized deployment

### 1. Install dependencies

```bash
npm install
```

### 2. Scrape card data

This downloads all 1081+ card images and data from [foursouls.com](https://foursouls.com) into a local SQLite database. Takes 10–20 minutes (rate-limited to be polite to their servers). Only needs to be run once.

```bash
make setup
# or: npm run scrape
```

Card images are saved to `data/cards/` and the database to `data/cards.db`.

### 3. Start development servers

```bash
make dev
# or: npm run dev
```

- Client: http://localhost:3000  
- Server API: http://localhost:3001

### 4. Production with Docker Compose

```bash
cp .env.example .env
# Edit .env if needed (ports, max rooms, etc.)
make setup        # scrape cards first
make up-build     # docker compose up --build
```

- Client: http://localhost:3000  
- Server: http://localhost:3001

## Configuration

Copy `.env.example` to `.env` and adjust:

| Variable | Default | Description |
|---|---|---|
| `SERVER_PORT` | `3001` | Port for the game server |
| `CLIENT_PORT` | `3000` | Port for the web client |
| `MAX_ROOMS` | `10` | Max concurrent game rooms |
| `VITE_SERVER_URL` | `http://localhost:3001` | URL the browser uses to reach the server |

## How to Play

1. One player clicks **Create Game** and shares the 5-letter room code.
2. Other players click **Join Game** and enter the code + their name.
3. Spectators can join with "Join as spectator" checked.
4. The host selects which card sets to include and clicks **Start Game**.

### In-game controls

- **Cards** — hover for a quick tooltip; click for full card details.
- **Attack** — click "Attack" on a monster slot during your turn. A dice roller appears. Click it to roll; the attack resolves automatically.
- **Loot cards** — click a card in your hand, then click "Play" (costs a loot play).
- **Items** — click your own items to toggle charged/deactivated state.
- **Coins / HP** — use the +/- buttons next to each stat.
- **Stack** — all loots and abilities appear in the stack panel. Click "Pass Priority" to pass. When all players pass, the top item resolves.
- **Hand sharing** — click "Share Hand" in your player area and pick a player.
- **Card Search** — click "Card Search" in the top bar to look up any card mid-game.

## Architecture

```
packages/
  scraper/    # One-time card scraper (Node.js + Cheerio)
  server/     # Game server (Express + Socket.io + SQLite)
  client/     # Web client (React + Vite + Tailwind + Zustand)
data/
  cards.db    # SQLite card database (generated by scraper)
  cards/      # Card images (generated by scraper)
```

### Server

- All game state lives in memory (`GameStore`).
- Each action is a socket event; the server mutates state and broadcasts a filtered `ClientGameState` to every player in the room.
- Hands are private: the server only sends hand card IDs to the player who owns them (or players they've shared with).
- Card effects are **manual** — the app tracks game state but players decide how card text is applied.

### Stack & Priority

Priority passes around the table. Any player with priority can play a loot, activate an ability, or pass. When all players pass in succession, the top item on the stack resolves. This matches the Four Souls extended rulebook exactly.

## Re-scraping / Updating Cards

To add new card sets or refresh card data:

```bash
npm run scrape
```

The scraper is incremental — it skips cards already in the database and only downloads new ones.
